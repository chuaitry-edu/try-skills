# 使用指南

## 基本流程

```
1. 有一个学习目标 → 用 try-plan 看看现不现实
2. 发现太难/太简单 → 用 try-stretch 定位
3. 不知道怎么学   → 用 try-method 配学法
4. 知道怎么学但不知道今天做什么 → 用 try-task
5. 学完就忘了      → 用 try-review
6. 不知道自己卡在哪 → 用 try（路由器）
```

## 常见场景

### 场景 A：完全零基础想学新东西

```
「我想学 Python，但完全零基础，每天 1 小时」
→ try-plan：算目标要多久
→ try-method：确定学法（动手/技能→ 先模仿）
→ try-task：今天写第一行代码
```

### 场景 B：学了一段时间卡住了

```
「我学 React 三个月了，还是不敢做项目」
→ try-stretch：定位（恐慌区）
→ 降难度：找一个更小的项目
→ try-task：今天写一个组件
```

### 场景 C：学完就忘

```
「我读完一本书，一周后全忘了」
→ try-review：主动回忆三遍法
→ 24 小时后：第一次复习
→ 3 天后：第二次
→ 1 周后：第三次
```

## 什么时候不用

- 没有学习目标 → 先确定目标
- 目标不确定 → 先用 try-plan 收窄
- 需要专业指导（医疗/健身/心理）→ 找专业人士
