# 目标分解法

> 出自 try-plan + try-task skill

## 大目标为什么常常失败

不是目标有问题，是**从目标到行动之间缺了一步**。

```
目标：「三个月学会 Python」
          ↓
大多数人直接跳到：「今天我要学 Python」
          ↓
然后：打开教程 → 太长了 → 明天再说 → 放弃
```

## 正确的分解

```
目标（3 个月学会 Python）
  → 阶段（基础语法 → Pandas → 项目）
    → 今天的唯一任务（写第一个脚本）
```

## 完成标准

每个阶段要有「做出了什么」才算过，不是「学了多少时间」。

```
❌ 「学完基础语法」（模糊）
✅ 「能写一个脚本读取 CSV 文件并输出统计结果」（可验证）
```

## 任务设计原则

- **一天只做一件事** — 做完是奖励，不是继续
- **任务要小到「肯定能做完」** — 感觉「好像太少了」是正确信号
- **有完成标志** — 「做到 XX 就算完成」
