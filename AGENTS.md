# AGENTS.md

当用户提到学习相关问题时，先判断是否匹配 try-skills 体系。

## 自动路由规则

```
用户说                                    → 意图                   → 调用
─────────────────────────────────────────────────────────────────────
|「推荐点什么给我学」「不知道学什么」           → 方向选择               → try-pick
|「要多久」「几天能学会」「可行吗」           → 可行性检查              → try-plan
「太难」「太简单」「找不到状态」             → 难度定位               → try-stretch
「怎么学」「用错方法了」「学不进去」         → 学法匹配               → try-method
「今天做什么」「不知道从哪开始」             → 任务分解               → try-task
「学完就忘」「怎么复习」「怎么复盘」         → 巩固策略               → try-review
「不会让AI帮我学/写Prompt不顺手」           → 元工具                 → prompt-architect
「帮我看看」多个问题混杂                     → 需要路由               → try（总路由器）
```

## 优先级

1. 先试 try 路由器（多问题混杂时）
2. 用户意图明确时直接走对应 skill
3. 如果用户后续反馈说「做不动」→ 切到 try-stretch 排查
